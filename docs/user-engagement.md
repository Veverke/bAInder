2. Message broader LinkedIn connections
3. Post announcement on LinkedIn

## Additional Recommendations

### Discovery & SEO
- **CWS listing quality**: compelling short description (first 132 chars are critical), screenshots with captions, a demo video — directly affects conversion from search impressions
- **CWS keywords**: embed relevant search terms naturally in the description (not stuffed)
- **GitHub repo**: polish the README — some users find extensions via GitHub search

### Community Channels
| Channel | Notes |
|---|---|
| Reddit | r/ChatGPT, r/productivity, r/AItools, r/artificial — post as a genuine user, be transparent you built it |
| Hacker News | "Show HN" post — skeptical audience but significant traffic spike and quality feedback |
| Product Hunt | Pick Tuesday–Thursday, prepare assets in advance; less traffic than before but still indexed and improves CWS trust signals |

### Content
- **Short demo video** (60–90 sec) showing a real use case — reusable across LinkedIn, Reddit, Product Hunt
- **Blog post** on dev.to or Medium: "How I built X" or "Why I built X" — technical audiences often install tools they discover this way

### Trust Signals
- Ask early users explicitly for CWS reviews — even 5–10 reviews change the listing's visibility tier
- Star the GitHub repo — social proof for developer-adjacent users

### Feedback Loop
- Set up a feedback channel (GitHub Discussions, Google Form, or Discord)
- Respond quickly to early CWS reviews/issues — signals active maintenance

### What to Defer
- Press outreach (TLDR, Ben's Bites newsletters) — better after you have reviews and a polished listing
- Paid promotion — not worth it until organic channels are validated

---

## Priority Order

1. Nail the CWS listing assets + get 5+ reviews from friends
2. Show HN post (high signal-to-noise feedback)
3. Reddit posts in relevant subs
4. LinkedIn post
5. Product Hunt when listing is solid

> **Single highest-leverage action most developers skip:** ask the first 20 users directly for a CWS review. It changes the listing's visibility tier.

## Demo Video Tools

### Screen Recording (no auto-voice)
| Tool | Price | Notes |
|---|---|---|
| OBS Studio | **Free** | Full control, no watermark, mic support — best quality |
| Loom | **Free tier** (limited) | Lowest friction, screen + face cam, instant shareable link |
| ScreenToGif | **Free** | Lightweight, GIF or video output, good for short loops |
| Cap | **Free / open source** | Open-source Loom alternative |

### With Auto-Voiceover / TTS Built In
| Tool | Price | Notes |
|---|---|---|
| Clipchamp | **Free** (built into Windows 11) | Microsoft TTS voices included |
| Camtasia | **Paid** (trial available) | TTS voice generation in recent versions; polished editor |

### AI Voice / Voiceover Specialists
| Tool | Price | Notes |
|---|---|---|
| Descript | **Freemium** (paid for AI voice) | Record screen, then overdub/replace with AI voice clone or TTS — strongest option for post-production voiceover |
| Synthesia | **Paid** | AI avatar + voice, very polished |
| HeyGen | **Paid** | Similar to Synthesia, AI presenter |
| Runway | **Paid** | Advanced video + AI; overkill for a simple demo |

### Recommended Workflow
> Record screen with **OBS Studio** or **Loom** (no voice), then bring the video into **Descript** to add AI voiceover. Best control + output quality at low cost.
